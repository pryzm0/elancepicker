Web Crawling & Data Scraping

 

Today the Web contains an enormous amount of valuable information which is provided in a semi-structured form, mostly suitable for viewing and reading by humans. While some apporaches of providing information for machines exist (like Semantic Web), it's still a crafty task when you need to collect some specific data from an arbitrary web service. Generally, you need a kind of robot which will sequentially access aimed resources, retrieve useful information and provide you processed and validated data in appropriate format on a regular basis.

Demand for data scraping is quite large now. Market research, lead generation, price comparision are very common tasks which arise just every day. There's also a plenty of tools for automated data retrieval, such as import.io or a number of personal desktop applications. But, because of the nature of the web, building the final solution with them still may be tricky. The best option is to hire a professional, who will develop a tool just for you needs.

As a specialist in web crawling, I'm interested in job offerings. Monitoring several freelance sites is a tedious work, which should be automated. Now then, I've built this service. It crawls major freelance sites and collects recent job offerings somehow related to web crawling. Besides that, this is my portfolio project.

Yep, I can scrape for you too! :-)
